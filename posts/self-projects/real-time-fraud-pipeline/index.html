The Architecture Matrix

To achieve sub-second latency, the system relies on a decoupled microservices architecture. Here is the core stack:

  • FastAPI: High-performance async ingestion.
  • Kafka: The central nervous system for event streaming.
  • Redis: Blazing fast caching and state management.
  • PostgreSQL: Persistent storage for long-term audit logs.
  • Docker: Containerized environments for isolated, reproducible deployments.

Initializing the Ingestion Node

Here is a snippet of how we configure the main entry point to handle incoming telemetry data and push it to our message broker. Notice how the syntax highlighting (which we can dial in later) sits inside the amber-glowing block.

from fastapi import FastAPI, BackgroundTasks
from kafka import KafkaProducer
import json

app = FastAPI(title="Anomaly Ingestion Node")
producer = KafkaProducer(
    bootstrap_servers=['kafka:9092'],
    value_serializer=lambda v: json.dumps(v).encode('utf-8')
)

@app.post("/api/v1/events")
async def ingest_event(payload: dict, background_tasks: BackgroundTasks):
    # Fire and forget to keep the API snappy
    background_tasks.add_task(producer.send, 'fraud_topic', payload)
    return {"status": "ACK", "message": "Event queued for processing"}
