System Architecture

The pipeline is designed to be highly concurrent, processing incoming payloads with sub-second latency. We chose a microservices approach to decouple the embedded data ingestion from the ML inference engine.

[!SYSTEM WARNING] The current staging environment has a strict rate limit of 500 req/sec. Pushing the stress test beyond this will result in a localized memory leak in the Redis cache buffer.

Deployment Roadmap

The build is currently in Phase 2. The legacy REST API has been deprecated and replaced entirely by gRPC streams.

  • Phase 1: Core hardware assembly and firmware flash.
  • Phase 2: Establish gRPC streams between Edge and Cloud.
  • Phase 3: Train and deploy the initial TinyML anomaly model.
  • Phase 4: Over-The-Air (OTA) update pipeline testing.
